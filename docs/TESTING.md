# RootA — раздел «Тестирование» (Часть 16)

> Документ для переноса в Notion / Buildin.  
> Репозиторий: `RootA` (Android, Kotlin, Jetpack Compose, Room).

---

## 1. Цель тестирования

Проверить готовность MVP мобильного приложения **RootA** к защите: основные пользовательские сценарии работают стабильно, данные сохраняются в локальной БД, бизнес-логика графа задач (зависимости, прогресс, циклы) корректна.

**Ссылка на сборку:** локальная debug-сборка из репозитория (`./gradlew assembleDebug`).

---

## 2. Тест-план

| Поле | Ответ |
|------|-------|
| **Название проекта** | RootA — графовая система задач для Android |
| **Цель тестирования** | Подтвердить работоспособность MVP перед защитой |
| **Что проверяется** | Онбординг, CRUD проектов/задач, граф, зависимости, прогресс, настройки, валидация форм, Room БД |
| **Что не входит в тестирование** | Облачная синхронизация, push-уведомления, мультиаккаунт, tablet-адаптация |
| **Виды тестирования** | Модульное, интеграционное, UI/E2E, smoke, ручное, приёмочное |
| **Тестовые данные** | Проект «Тестовый проект», задачи с приоритетами, дедлайн `01:06:2026`, пустые/некорректные поля |
| **Критерии начала** | Приложение собирается, основные экраны реализованы |
| **Критерии завершения** | Все Must Have из ТЗ проверены; критические ошибки закрыты или задокументированы |
| **Основные риски** | Нет эмулятора для UI-тестов; silent-валидация пустых полей в UI |

## 10. Итоговый отчёт о готовности MVP

### Что работает
- Онбординг и навигация по вкладкам
- CRUD проектов и задач с сохранением в Room
- Граф задач: узлы, зависимости, авто-раскладка, zoom
- Валидация дедлайна и блокировка циклов / преждевременного завершения
- Расчёт прогресса по проекту и глобально
- Очистка всех данных

### Что исправлено в рамках тестирования
- Добавлены модульные тесты для `DateFormatters`, `AddDependencyUseCase`, `ChangeTaskStatusUseCase`, `CreateProjectUseCase`, `CreateTaskUseCase`
- Добавлены интеграционные тесты слоя данных
- Расширен UI-тест основного сценария

### Что осталось (не блокирует защиту)
- BUG-01, BUG-02: показывать сообщение при пустом названии проекта/задачи
- Скриншоты и видео для Notion — оформить вручную на эмуляторе

### Вывод
**MVP RootA готов к демонстрации.** Критических ошибок не обнаружено. Основной пользовательский сценарий подтверждён автоматическими и ручными проверками. Два minor-бага зафиксированы и могут быть исправлены в следующей итерации.
